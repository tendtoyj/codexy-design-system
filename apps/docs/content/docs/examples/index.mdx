Examples는 **완성된 화면**을 보여주는 자리다. Patterns가 "이 상황엔 이렇게"라면, Examples는 "실제로 이렇게 생긴다"를 보여준다.

## 패턴과 어떻게 다른가

| 축 | Patterns | Examples |
| --- | --- | --- |
| 단위 | 조합 단위 (Dialog + Form) | 화면 단위 (설정 페이지, 채팅 화면) |
| 목적 | 의사결정 가이드 | 시각·인터랙션 레퍼런스 |
| 재사용 | 패턴 자체를 차용 | 부분만 발췌해 차용 |

## 작성 원칙

- **풀스크린 미리보기** — 실제 밀도·여백을 그대로 보여줌. 필요 시 `app/preview/` 라우트 재활용
- **조합 표기** — 화면을 구성하는 패턴/컴포넌트 목록을 상단에 명시
- **데이터는 더미** — 실제 API 의존 없이 정적 데이터로

## 현재 상태

뼈대만 잡힌 상태. 첫 예시(채팅 화면 등)는 이후 별도 PR 로 추가된다.
